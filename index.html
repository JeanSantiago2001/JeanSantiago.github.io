Soy Jean Paul Santiago Tumialán y estudio la Carrera de Ingeniería de Sistemas en la Universidad Nacional del Centro del Perú. Destaco por la capacidad para resolver desafíos complejos mediante el desarrollo de software y la gestión eficiente de bases de datos. Además de mis habilidades técnicas, me destaco por mi capacidad para trabajar en equipo, comunicarme efectivamente y colaborar para lograr objetivos compartidos. Mi pasión por la tecnología y mi constante búsqueda de conocimiento me posicionan como un estudiante comprometido con el aprendizaje continuo y preparado para enfrentar los desafíos en la industria tecnológica en constante evolución.

1ER CONSOLIDADO

Durante la primera evaluación del curso de desarrollo de aplicaciones web, hemos logrado avances notables. En términos de HTML y CSS, aprendimos a crear páginas web con diseño responsivo, adaptándose de manera óptima a diversos dispositivos y tamaños de pantalla, también aprendimos a usar bibliotecas como Bootstrap y sus colección de plantillas. Además, adquirimos conocimientos en JavaScript, lo que nos permitió animar objetos y elementos en las páginas web, proporcionándoles interactividad y dinamismo. Exploramos el uso de Canvas, lo que nos brindó la capacidad de dibujar objetos y figuras. Finalmente, nos adentramos en el mundo de Node.js, aprendiendo a desarrollar aplicaciones web que se ejecutan en el lado del servidor. En resumen, los resultados obtenidos en esta primera evaluación fueron satisfactorios y nos motivan a continuar nuestro aprendizaje con dedicación y esfuerzo.

Implementé alertas visuales con SweetAlert2 y utilicé useContext para una transmisión eficiente de valores de estado entre componentes, simplificando la estructuración del proyecto.

El desarrollo del custom hook useForm se centró en la modularización eficiente de la lógica de formularios, proporcionando una comprensión práctica de su aplicación. La organización de componentes en archivos separados se llevó a cabo conscientemente, aplicando buenas prácticas de desarrollo. El uso de useRef y Axios se aprendió mediante la aplicación directa en situaciones prácticas, como el acceso directo a nodos del DOM y la conexión a APIs externas. La implementación de alertas visuales y el uso de useContext se abordaron con el objetivo de mejorar la estructuración y comunicación entre componentes.

¿QUÉ APRENDÍ?

Durante esta semana, adquirí conocimientos significativos sobre Next.js, un marco de React para aplicaciones web de pila completa. Exploré el concepto de Single Page Applications (SPAs) y comprendí cómo Next.js aborda las desventajas relacionadas con el SEO, ofreciendo soluciones para mejorar la indexación en motores de búsqueda. Aprendí a utilizar Server Components en Next.js para realizar operaciones del lado del servidor, como consumir la API de Rick and Morty. Además, me familiaricé con la implementación de Fetch en Server Components, la descarga de proyectos desde GitHub, la instalación de dependencias y la creación de funciones para obtener y renderizar datos de manera eficiente.

¿COMO APRENDÍ?

El aprendizaje se llevó a cabo a través de una combinación de práctica activa y comprensión teórica. Descargué proyectos desde GitHub y configuré entornos, aplicando los conocimientos prácticos de configuración inicial. La instalación de dependencias se realizó mediante comandos como npm install, consolidando la comprensión de las herramientas utilizadas en el entorno Next.js. La creación de funciones para solicitudes a la API de Rick and Morty, junto con el uso de Server Components, se implementó prácticamente, fortaleciendo mi comprensión de cómo realizar operaciones del lado del servidor. La aplicación de la técnica de vinculación de enlaces y el uso de Next Image para mostrar imágenes de personajes enriquecieron la experiencia de aprendizaje mediante la aplicación de conceptos teóricos en un contexto práctico. En resumen, aprendí de manera activa y práctica, combinando la aplicación directa de procedimientos con una comprensión sólida de los conceptos fundamentales de Next.js.

Implementé la gestión de sesiones, explorando conceptos clave como el proveedor de sesiones y el uso de middleware en NextAuth.

¿COMO APRENDÍ?

El aprendizaje se llevó a cabo mediante la práctica activa y la aplicación directa de los conceptos en un proyecto concreto. La instalación y configuración del backend se realizó de manera práctica, ejecutando comandos y comprendiendo el flujo del sistema. La exploración y configuración de NextAuth se basó en la documentación oficial, aplicando los procedimientos paso a paso. La instalación de Bootstrap y su implementación en el frontend se realizó de manera consciente, mejorando la interfaz de usuario de la aplicación. La creación de componentes y la configuración de middleware para gestionar rutas protegidas se llevaron a cabo en un contexto práctico, fortaleciendo la comprensión de la autenticación y autorización en Next.js. En resumen, aprendí activamente, aplicando conceptos teóricos en un proyecto práctico y experimentando con la configuración y uso de herramientas clave como NextAuth y Bootstrap.

Exploré características y módulos esenciales de Spring Framework, como Spring MVC para desarrollo web. Profundicé en el funcionamiento del Spring Container, clave para la gestión de objetos (beans) y asimilé conceptos como el Principio Hollywood y el término "beans". Este conocimiento sienta una base sólida para el desarrollo eficiente de aplicaciones empresariales en Java con Spring.

¿COMO APRENDÍ?

El aprendizaje se basó en una combinación de teoría y práctica. La configuración de Spring mediante archivos XML se llevó a cabo aplicando procedimientos prácticos, donde definí objetos y dependencias de manera estructurada. La creación de clases y objetos, como el ejemplo de la clase "Estudiante.java", proporcionó una comprensión práctica de la representación de componentes en una aplicación empresarial. La utilización del contenedor de IoC de Spring se aprendió aplicándolo a la creación, configuración y gestión de objetos, fortaleciendo mi comprensión sobre modularidad y reutilización de código. La aplicación del patrón de Inyección de Dependencia mejoró la mantenibilidad de la aplicación al separar la configuración de la lógica de negocio. La integración de tecnologías como JPA, Hibernate, Struts y JSF se exploró, proporcionando una visión más amplia sobre la flexibilidad y la expansión de la aplicación al integrar diferentes frameworks y tecnologías. En resumen, aprendí mediante la aplicación práctica de conceptos en un entorno Java con Spring, fortaleciendo mi comprensión teórica con la experiencia directa en la configuración y desarrollo de aplicaciones empresariales.

¿QUÉ APRENDÍ?

Asimismo, desarrollé competencias esenciales en tecnologías web, como JavaScript, manipulación del DOM, validación de formularios y consumo de APIs externas con Axios.
