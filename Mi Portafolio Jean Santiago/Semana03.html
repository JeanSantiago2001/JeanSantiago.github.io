¿Qué Temas Aprendió?

DESARROLLO DE APLICACIONES WEB

Durante la semana de estudio, se profundizó en JavaScript y su aplicación en el desarrollo web interactivo y dinámico. Se exploraron las funciones esenciales de este lenguaje, que posibilitan la creación de interactividad en las páginas web mediante validaciones, animaciones y la manipulación en tiempo real del contenido. Se hizo hincapié en el uso de eventos para responder a las acciones del usuario, como clics o movimientos del ratón, mejorando así la experiencia del usuario en la página.

Además, se profundizó en la manipulación del DOM, que representa estructuralmente el contenido de una página web. JavaScript proporcionó herramientas para acceder y modificar dinámicamente el contenido, la estructura y los estilos del DOM, lo que permite crear efectos visuales, actualizaciones en tiempo real y mejorar la usabilidad del sitio.

También se abordó la validación de formularios, donde se exploraron técnicas para verificar que los datos ingresados cumplan con criterios específicos, como campos obligatorios o formatos de fecha. Esta práctica es esencial para asegurar la precisión de los datos y proporcionar una experiencia de usuario sin contratiempos.

En resumen, se adquirió un conocimiento exhaustivo de JavaScript y su aplicación práctica en el desarrollo web interactivo y funcional, permitiendo mejorar la experiencia del usuario y la funcionalidad de las páginas web.

JavaScript

JavaScript es un lenguaje de programación interpretado y orientado a objetos, que se utiliza para desarrollar aplicaciones web interactivas. A medida que el lenguaje evolucionó, los desarrolladores establecieron bibliotecas, marcos y prácticas de programación, lo que permitió utilizar JavaScript tanto en el lado del cliente como en el lado del servidor.
JavaScript se utiliza en conjunto con HTML, y su código puede colocarse cerca del final de la página para asegurarse de que se cargue después del HTML sobre el que debe trabajar. Aunque JavaScript fue diseñado originalmente para su uso en navegadores web, su popularidad ha llevado a su adopción en otros entornos, como entornos JavaScript del lado del servidor.

Eventos

Son incidencias o cambios significativos en un sistema que desencadenan una respuesta por parte de la aplicación. Estos eventos pueden ser generados por el usuario, como hacer clic en un botón, o por el sistema, como la carga completa de una página. El manejo adecuado de eventos es fundamental para proporcionar experiencias de usuario eficientes y fluidas en aplicaciones web y móviles.

Manipulación del DOM

La manipulación del DOM (Document Object Model) es un concepto fundamental en el desarrollo de aplicaciones web. El DOM es una interfaz que permite a los programas de computadora acceder, manipular e incluso eliminar objetos del árbol de un documento, como un archivo HTML. El DOM se representa como una estructura de árbol, donde cada elemento HTML es un nodo en el árbol. Los nodos pueden tener relaciones padre-hijo, hermano-hermano y ancestro-descendiente, lo que permite a los desarrolladores acceder y manipular diferentes partes de un documento web.

Validación de formularios

La validación de formularios en el desarrollo de aplicaciones web se refiere al proceso de verificar y garantizar que los datos ingresados por los usuarios en un formulario cumplan con ciertos criterios predefinidos. Esta validación puede ocurrir en el lado del cliente, es decir, En el navegador web del usuario, o en el lado del servidor, donde se procesan los datos enviados por el formulario.

  4. Manipulación del DOM con JavaScript: Se adquirieron conocimientos sobre la manipulación del DOM con JavaScript, lo que posibilita cambiar dinámicamente el contenido, la estructura y los estilos de una página web.
  5. Validación de formularios: Por último, se abordó la validación de formularios en JavaScript, aprendiendo a utilizar las funciones de validación para asegurar que los datos ingresados por el usuario cumplan con los criterios requeridos.

RESULTADOS

DESARROLLO DE APLICACIONES WEB

  1. A partir de la respuesta HTML del servidor, se construyó el Document Object Model (DOM), que sirvió para representar la estructura y el contenido de la página web.
  2. Se desarrolló el CSS Object Model (CSSOM) para representar los estilos CSS empleados en la página. Esto posibilitó la aplicación de estilos específicos a cada elemento del DOM.
  3. Se combinaron el DOM y el CSSOM para formar el Render Tree, que mostraba los elementos visibles en la página junto con sus estilos aplicados. Esta fusión permitió determinar qué elementos serían mostrados en la pantalla y cómo se verían.
  4. Se llevó a cabo el cálculo del layout, en el cual se determinaron la posición y el tamaño exactos de cada elemento en la página. Esto implicó establecer las dimensiones y la ubicación de los elementos en relación con otros elementos y el contenedor principal.
