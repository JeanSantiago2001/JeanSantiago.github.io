DESARROLLO DE APLICACIONES WEB

Durante esta semana, se adquirió comprensión sobre la importancia de la plataforma web abierta como estándar para el desarrollo de aplicaciones en línea. Se examinó la estructura y el contenido de las páginas web a través de HTML, así como su aspecto y presentación mediante CSS. Además, se proporcionó una introducción al diseño y la disposición de las páginas utilizando técnicas como Flexbox y Grid Layout. Estos aprendizajes nos proporcionaron una base sólida para la creación y el diseño de aplicaciones web.

Open Web Platform

La Plataforma Web Abierta es una recopilación de tecnologías establecidas como estándares abiertos por entidades como el Consorcio World Wide Web (W3C) y otros organismos de estandarización, como el Grupo de Trabajo de Tecnología de Aplicaciones de Hipertexto Web, el Consorcio Unicode y el Internet Engineering Task Force. Este conjunto de tecnologías permite a los desarrolladores crear contenido que sea interoperable en un entorno unificado y cohesivo. Algunas de las tecnologías incluidas en la Plataforma Web Abierta son:

  • HTML (HyperText Markup Language): Lenguaje de publicación de la Web
  • CSS (Cascading Style Sheets): Lenguaje de hojas de estilo utilizado para describir la presentación de un documento HTML.
  • SVG (Scalable Vector Graphics): Formato de gráficos vectoriales basado en XML
  • Web Workers: API para la ejecución de scripts en segundo plano
  • Geolocation API: API para obtener la ubicación del usuario
  • DOM Events: API para la manipulación de eventos en el Document Object Model
  • Cross-Origin Resource Sharing: Mecanismo para permitir que los recursos de un origen sean utilizados por otro origen

Unidades de medida en CSS

Las unidades de medida en CSS se utilizan para dimensionar elementos en una página web. Estas unidades pueden ser absolutas o relativas, y existen varias opciones para cada tipo. A continuación se muestra una lista de las unidades de medida más comunes en CSS:

  • Unidades de medida absolutas:
    • in: pulgadas, equivalentes a 2.54 cm.
    • cm: centímetros.
    • mm: milímetros.
    • q: cuarto de milímetro, 1q = 0.248 mm.
    • pt: puntos, 1pt = 1/72 de pulgada o 0.35 mm.
    • pc: paicas, 1pc = 12 puntos.
  • Unidades de medida relativas:
    • em: relativa al tamaño del elemento padre.
    • rem: relativa al tamaño base del documento HTML.
    • ex: relativa a la altura de una "x" minúscula.
    • ch: relativa al ancho de un carácter "0" (cero).
    • vw: relativa al 1% del ancho de la ventana.
    • vh: relativa al 1% de la altura de la ventana.
    • vmin: relativa al 1% del tamaño mínimo entre el ancho y la altura de la ventana.
    • vmax: relativa al 1% del tamaño máximo entre el ancho y la altura de la ventana.
  • Unidades de medida adicionales:
    • %: porcentaje, relativo al valor del elemento padre.
    • fr: fracciones, se utiliza en diseños de cuadrícula (grid layouts).
    • auto: el navegador calcula automáticamente el valor.
    • fit-content: el tamaño se ajusta al contenido del elemento.
    • min-content: el tamaño mínimo necesario para mostrar todo el contenido.
    • max-content: el tamaño máximo posible para mostrar todo el contenido.

CSS (Cascading Style Sheets)

CSS se utiliza en conjunto con HTML para crear páginas web visualmente atractivas y bien estructuradas. Los desarrolladores web utilizan CSS para definir la apariencia de los elementos HTML, como el tamaño y el color de la fuente, el color de fondo, el espaciado entre líneas, el espaciado entre elementos, etc. Al separar la presentación del contenido, CSS permite a los desarrolladores web crear páginas web más eficientes y flexibles

HTML (HyperText Markup Language)

La estructura de una página web se basa en HTML, que proporciona el esqueleto de la página. Sin embargo, HTML por sí solo no es suficiente para hacer que una página web sea visualmente atractiva e interactiva.

PROCEDIMIENTOS

DESARROLLO DE APLICACIONES WEB

1. Creación del Document Object Model (DOM):

Se genera el DOM a partir del código HTML recibido del servidor. Esto involucra la conversión del HTML en una estructura de árbol que representa la jerarquía de elementos de la página.

2. Creación del CSS Object Model (CSSOM):

Se crea el CSSOM, que es una representación de las reglas CSS utilizadas en la página. Esto implica analizar y procesar las reglas CSS para determinar los estilos aplicados a cada elemento del DOM.

3. Combinación del DOM y el CSSOM en el Render Tree:

Se combinan el DOM y el CSSOM para formar el Render Tree. Este árbol es una representación de los elementos visibles en la página y sus estilos aplicados. Aquí se determina qué elementos se mostrarán en la pantalla y cómo se verán.

4. Cálculo del layout:

Una vez que se tiene el Render Tree, se procede al cálculo del layout. Es decir, se determina la posición y el tamaño exactos de cada elemento en la página. Esto implica establecer las dimensiones y la ubicación de los elementos en relación con otros elementos y con el contenedor principal, para definir cómo se visualizará la página en pantalla.

RESULTADOS

DESARROLLO DE APLICACIONES WEB

1. Generación del DOM a partir del HTML del servidor:

El DOM (Document Object Model) se crea a partir de la respuesta HTML del servidor. Esto permite representar la estructura y el contenido de la página web como un árbol de nodos que representan cada elemento HTML.

2. Creación del CSSOM para representar los estilos CSS:

El CSSOM (CSS Object Model) se forma para representar los estilos CSS utilizados en la página. Esto posibilita la aplicación de estilos específicos a cada elemento del DOM, definiendo cómo se verán en la interfaz.

3. Combinación del DOM y el CSSOM en el Render Tree:

La unión del DOM y el CSSOM da lugar al Render Tree. Esta estructura de árbol muestra los elementos visibles en la página junto con sus estilos aplicados. El Render Tree determina qué elementos se mostrarán en la pantalla y cómo se visualizarán, considerando la combinación de la estructura HTML y los estilos CSS.
